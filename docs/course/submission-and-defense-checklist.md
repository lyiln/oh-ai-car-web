# 课程提交与答辩清单

## 课程提交物

- [ ] 每名成员 10 天日报，含当天实际工作、证据链接和次日计划。
- [ ] 需求分析报告：五项基本需求、功能与非功能需求、范围边界和概要设计。
- [ ] 中期检查 PPT：当前完成度、功能演示、风险、剩余计划和成员分工。
- [ ] 最终答辩 PPT：立项、管理、设计实现、成果展示和贡献度。
- [ ] 项目开发与测试报告：模块启动方式、测试环境、结果、已知限制和真车记录。
- [ ] 项目使用手册：环境要求、启动顺序、配置、操作、安全停止和故障处理。
- [ ] GitHub/Gitee 代码链接、固定提交或标签、演示录屏及素材索引。

## 最终答辩

- [ ] 展示需求如何细化为功能和非功能需求，以及概要设计。
- [ ] 展示甘特图或燃尽图、实际进度和延期风险。
- [ ] 单独一页列出成员分工与可核验贡献。
- [ ] 每名成员讲解自己实现的代码、测试或系统集成，而非只讲文档。
- [ ] 演示视频覆盖稳定流程；现场演示只执行已在安全环境验证的功能。
- [ ] 准备真车不可用时的离线演示：Fake TCP、自动化测试、录屏和明确的未验证声明。
- [ ] 全体成员到场发言；材料在答辩结束后两日内以小组为单位提交。
