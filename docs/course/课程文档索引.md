# 课程文档索引

本目录服务于整个五人小组的课程交付；`specs/` 仍只定义 Web 控制端的产品需求，`docs/reference/` 仍只保存原车端源码证据。三者不得互相替代。

| 文档 | 用途 | 何时更新 |
| --- | --- | --- |
| [`../../课程状态.md`](../../课程状态.md) | 课程状态首页和关键结论 | 每次中期或最终验收前 |
| [五人小组交付执行手册.md](五人小组交付执行手册.md) | 五人执行方案、模块验收、进阶切换条件 | 分工或项目范围变化时 |
| [基本需求证据矩阵.md](基本需求证据矩阵.md) | 每项必做需求的负责人和证据链接 | 每天由负责人更新 |
| [课程要求与任务书映射.md](课程要求与任务书映射.md) | 课程 PDF、任务书与小组交付的对应关系 | 教师澄清或任务书变化时 |
| [提交与答辩清单.md](提交与答辩清单.md) | 中期、最终答辩和课程提交物收口 | 每次汇报前 |
| [Web控制模块答辩材料.md](Web控制模块答辩材料.md) | Web 模块讲解、已验证行为和真车边界 | Web 行为或真车结果变化时 |

## 文档规则

- “已验证”必须附带可定位的代码、命令、截图、日志或视频；计划、口头描述和 Fake TCP 测试不能证明真车行为。
- 课程评分依据以课程简介 PDF 和任课教师要求为准；任务书用于技术拆解与验收材料设计。
- 将 Web 代替 OpenHarmony APP 的课程认可状态必须写入证据矩阵，不能仅在答辩时口头说明。
- 每名成员应维护自己的日报和证据链接；文档管理员只能汇总，不能代替实际贡献。
