# Web 控制模块答辩材料

## 定位

本模块拟对应课程基本需求“开发 APP 控制小车运动”。浏览器界面承担操作端，Node 网关只监听本机并将经过校验的高层命令写入小车 TCP 服务；它不复制或依赖原 OpenHarmony 工程。课程任务书以鸿蒙 APP 为训练材料，因此中期前必须取得教师对 Web 交付形态的认可并记录。

## 可讲解实现

- 前端提供地址配置、按钮、摇杆、四轮速度、媒体、循迹和视频入口。
- 网关限定 `127.0.0.1:8787`，限制本地 Origin，拒绝原始 TCP 指令透传，并保证单浏览器控制权。
- 浏览器状态同时展示 TCP 是否连接、当前页面是否拥有控制权和是否可接管，避免观察页面误操作。
- 显式断开、重连、控制页关闭和网关关闭均尝试先发送 Stop；Stop 写入失败会关闭 TCP 并报告失败。
- 共享编码器覆盖 `10`、`15`、`21`、`60-64` 命令，负速度按既有源码规则处理；编码、假 TCP 网关和前端交互均有自动化测试。

## 必须如实说明的边界

- `$011503011A#` 与 `$011504011B#` 存在来源冲突。当前实现采用 ArkTS 源码兼容基线，尚未由真车确认。
- 成功结果表示网关 TCP 写入成功，不表示车辆 ACK 或执行成功。
- iframe 页面加载不表示视频流健康。
- 真车可用后，按 `docs/flows/web-control-real-car-validation.md` 逐项验证并记录；未记录前状态只能写“待验证”。
