# 课程要求与任务书映射

## 依据优先级

| 来源 | 决定什么 | 本项目如何使用 |
| --- | --- | --- |
| `2026-实训 - 1. 课程简介.pdf` | 五项基本需求、90 分基础构成、10 分加分项、提交物和答辩形式 | 评分与范围的最高依据 |
| `五人三天详细任务计划书.md` | 五人角色、每阶段最低产出、中期展示顺序 | 小组日常执行基线 |
| 各技术任务书 | 模块操作、提交物、验收标准和安全限制 | 各负责人实现与取证依据 |
| 本仓库 Spec Kit 与源码 | Web 控制端功能、协议、安全和测试边界 | 仅用于 Web 模块实现 |
| 任课教师当面要求 | 对 APP 形态、进阶项和小组分级的最终解释 | 记录后优先于本目录推断 |

## 五项基本需求的落地映射

| 课程基本需求 | 任务书依据 | 小组最低可验收结果 |
| --- | --- | --- |
| ROS2 核心开发与分布式通信 | 雷达任务书的 `sllidar_ros2`、`/scan`、`/cmd_vel`、`rqt_graph`；视觉任务书的 RGB、深度、`/Current_point`、`/cmd_vel` | 至少一个 ROS2 工作空间可构建；展示节点、话题和消息流；保存 `rqt_graph`、终端日志和启动命令 |
| SLAM 建图与自主导航 | SLAM 任务书的 Gmapping、地图保存、AMCL、Navigation2、单点/多点导航 | 保存 `pgm/yaml` 地图；展示 `/scan`、`/map`、定位和目标导航证据；实车不可用时明确标记为仿真或待验证 |
| AI 边缘推理与多传感器融合 | 视觉任务书的 Astra RGB/深度、YOLOv5、Jetson 推理、网络推理和 ROS2 追踪 | 有模型推理结果；边缘端或教师认可的等价环境记录；展示 RGB 检测与深度距离如何形成位置/控制数据 |
| 云平台集成与 CI/CD 流水线 | 环境感知任务书的 NB-IoT 连接华为云、WiFi；课程 PDF 的云平台与 CI/CD | 至少一条云端数据或连接记录，加上一条可重复运行的 CI 流水线，输出构建/测试结果；两者缺一不可 |
| 开发 APP 控制小车运动 | 鸿蒙 APP 任务书的 UI、摇杆、指令编码和 TCP；当前 Web Spec Kit | 操作界面、控制命令、Stop 安全处理、TCP 写入记录和真车验证。Web 替代鸿蒙 APP 需要教师确认 |

## 必须澄清的事项

在最近一次教师沟通中记录以下问题与答复：

1. 浏览器 Web 控制端是否可作为“开发 APP 控制小车运动”的合格实现，是否还需保留鸿蒙端演示。
2. “云平台集成与 CI/CD”对云端平台、数据类型和流水线平台是否有指定要求。
3. “跨平台安全接管与控制链路系统优化”能否计入进阶项，教师希望看到哪些量化证据。

答复应填写到证据矩阵末尾的“教师确认/备注”记录或项目会议记录中。
