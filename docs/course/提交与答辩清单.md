# 课程提交与答辩清单

## 中期检查

- [ ] 项目主题、五项基本需求、总体架构和五人分工。
- [ ] 每位成员一页“已完成 + 证据 + 未解决风险”，并准备 1-2 分钟讲解。
- [ ] 展示至少一条每个模块的截图、日志、仿真结果或录屏；没有实车结果时明确说明。
- [ ] 展示进度表、剩余计划和教师需要确认的问题：Web APP 认可、云/CI 标准、进阶项认可。

## 最终提交物

- [ ] 每名成员 10 天日报，含当天实际工作、证据链接和次日计划。
- [ ] 需求分析报告：五项基本需求、功能与非功能需求、范围边界和概要设计。
- [ ] 中期检查 PPT 和最终答辩 PPT。
- [ ] 开发与测试报告：模块启动方式、测试环境、结果、已知限制和真车记录。
- [ ] 项目使用手册：环境要求、启动顺序、配置、操作、安全停止和故障处理。
- [ ] GitHub/Gitee 代码链接、固定提交或标签、演示录屏、截图、日志、模型、地图和配置索引。

## 最终答辩

- [ ] 用一张架构图说明五项基本需求如何组成同一项目。
- [ ] 展示甘特图或燃尽图、实际进度、风险和补救结果。
- [ ] 单独一页列出成员分工与可核验贡献；每个人讲解自己的代码、测试或集成。
- [ ] 演示视频覆盖稳定流程；现场演示只执行安全条件下验证过的功能。
- [ ] 真车不可用时，展示 Fake TCP、自动化测试、仿真、录屏和明确的未验证声明。
- [ ] 进阶项单独展示教师认可、问题、优化方案、量化/场景证据和局限。
- [ ] 全体成员到场发言；答辩结束后两日内以小组为单位提交。
