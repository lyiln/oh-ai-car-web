# AI Context

## Confirmed Project Background
- The project is named `oh-ai-car-ros-app` and is described by `readme.md` as a smart car ROS integration version.
- The application is implemented with OpenHarmony/ArkTS project structure: `entry/`, `Rocker/`, `AppScope/`, `hvigor/`, `oh-package.json5`.
- The app controls a smart car through TCP commands and displays video through an HTTP page.

## Important Directories
- `entry/src/main/ets/pages/`: application pages, including network setup, home selection, remote control, and mecanum wheel control.
- `entry/src/main/ets/components/`: UI components for buttons, rocker control, and video.
- `entry/src/main/ets/CarUtill/`: car command API, encoding, and direction enum.
- `entry/src/main/ets/tcp/`: TCP connection and send/receive utilities.
- `Rocker/src/main/ets/components/`: reusable rocker component and drawing utilities.
- `doc/`: original project documentation, prototypes, and ROS API notes.
- `docs/`: AI collaboration documentation created for analysis and future planning.
- `tasks/`: short-lived task plans, fix plans, and execution notes.

## Confirmed Runtime Flow
- `EntryAbility.ets` initializes preferences and loads `pages/NetworkSettings`.
- `NetworkSettings.ets` reads/saves IP, TCP port, and video port preferences.
- On successful TCP connection, `NetworkSettings.ets` navigates to `pages/Index`.
- `Index.ets` routes to `pages/MecanumWheel` and `pages/RemoteControl`.
- `RemoteControl.ets` provides button/rocker control plus video and tracking controls.
- `MecanumWheel.ets` sends four-wheel speed updates.

## Confirmed Network Defaults
- Default car IP in source: `192.168.1.11`.
- Default TCP control port: `6000`.
- Default video port: `6500`.
- Video URL pattern: `http://<ip>:<videoPort>/index2`.

## Future Web Direction
- The default Web architecture is frontend plus gateway.
- The browser should connect to the gateway through HTTP/WebSocket.
- The gateway should translate Web commands into the same TCP strings currently generated by `CarEncode.ets`.
- The first Web implementation should prioritize network config, button control, rocker control, video preview, photo/record commands, and tracking commands.
