## 2. P0 核心页 Figma 提示词

### 2.1 工作台 `/dashboard`

```
巡牌云工作台首页。顶部 4 个 KPI 卡片：在线设备数、今日巡逻次数、待审核数、违规车辆数。
中部左侧「快捷操作」：添加设备、进入控制台、发起巡检、查看报告。
中部右侧「最近巡逻记录」表格（时间、设备、路线、状态、报告入口）。
底部「今日告警」列表（违规停放高优先级优先，红色标签）。
深色 TO B 风格，数据可视化简洁，不要花哨图表。
```

### 2.2 设备管理 `/fleet`

```
设备管理页——管理巡检小车 iCar 设备。
顶部：标题「设备管理」+ 主按钮「添加设备」。
设备卡片列表/表格，字段：设备名称、设备编号、IP 地址、TCP 端口、在线状态、最近巡逻时间、操作（连接/编辑/删除）。
空状态：插画+文案「暂无设备，请先添加巡检小车」+ 添加按钮。
「添加设备」抽屉/弹窗表单：设备名称*、设备编号*、IP*、TCP端口(默认6000)、Bridge地址、备注；保存后回到列表，已添加设备持久保留。
每行有「设为当前设备」按钮，选中后顶部栏显示当前设备名。
状态：在线(绿)/离线(灰)/巡逻中(蓝)/故障(红)。
```

### 2.3 控制台 `/console`

```
控制台页，左右分栏（6:4 或 5:5）。
左侧：实时地图区域（占主视觉）。
  - 底图：小区 SLAM 地图（灰度栅格或简化矢量）
  - 实时显示当前巡检小车位置（蓝色箭头图标+坐标）
  - 显示巡逻轨迹虚线（本次连接期间）
  - 地图右上角：缩放、复位、跟随小车开关
  - 地图上叠加航点标记、禁停区半透明红色多边形
右侧：控制面板（复用遥控能力）。
  - 顶部：当前设备名、连接状态、电量/信号（占位）
  - Tab：方向按钮 / 摇杆 / 麦轮
  - 视频预览窗口（16:9）
  - 拍照、停止、急停按钮（急停红色醒目）
  - 底部安全提示：「松开摇杆自动停止」
未选设备时显示拦截页：「请先在设备管理中添加并选择巡检小车」。
全部中文，深色主题，地图区域边框 #2D3A4F。
```

### 2.4 巡检任务 `/patrol/tasks`

```
巡检任务下发页。
左侧任务配置卡片：
  - 选择设备（下拉，来自已添加小车）
  - 选择路线/航点方案（route_morning_a 等）
  - 班次（早/午/晚）
  - 白名单检查状态（已导入 X 条，绿灯才可启动）
  - 禁停区预览（小地图缩略）
  - 主按钮「开始巡检」/ 次按钮「停止巡检」
右侧：任务状态面板（状态机：空闲/导航中/到点识别/已完成）
  - 当前航点、进度 X/Y、已识别车辆数
  - 实时事件流（最新 10 条）
依据立项：同一时刻仅 1 个任务；白名单为空禁止启动。
```

### 2.5 巡逻记录 `/patrol/records` + 详情

```
巡逻记录列表页。
筛选：日期范围、设备、路线、状态（进行中/已完成/失败）。
表格列：任务ID、巡逻小车、开始时间、结束时间、航点完成率、识别事件数、违规数、报告、状态。
操作：查看详情、下载报告。
---
巡逻记录详情页（子页）：
  顶部任务概览卡片（设备、路线、耗时、航点完成率）
  Tab1「事件时间线」：每条识别记录（车牌、判定类型、航点、时间、缩略图）
  Tab2「轨迹地图」：本次巡逻轨迹叠加 SLAM 地图
  Tab3「巡逻报告」：嵌入 HTML 报告预览 + 下载 CSV/ZIP
  Tab4「处置状态」：已确认/误报/待处理统计
```

## 3. 运营页 Figma 提示词

### 3.1 全局大地图 `/map`

```
全局地图页——小区运营核心视图，全宽布局。
主区域：可缩放/平移大地图（SLAM map 或卫星图占位）。
工具栏：
  - 「绘制禁停区」：多边形绘制模式，保存 ROI
  - 「编辑禁停区」：选中已有区域修改/删除
  - 图层开关：航点/禁停区/违规车辆/巡逻轨迹
地图上元素：
  - 禁停区：红色半透明多边形 + 区域名称标签
  - 违规车辆：红色车辆图标 Marker，聚合显示数量
  - 航点：蓝色圆点
  - 巡检小车：实时位置（若在巡逻）
交互：点击违规 Marker 弹出侧边详情抽屉——
  车牌、判定类型、违规时间、所在禁停区、航点、置信度、证据截图、处置建议、按钮「去审核」。
右下角图例说明。中文标注。
```

### 3.2 违规车辆 `/violations`

```
违规车辆总览页，表格+筛选为主。
筛选：时间、禁停区域、设备、处置状态。
表格列：车牌、违规类型（禁停/疑似外来）、位置/航点、禁停区名称、发现时间、关联巡逻任务、证据图、优先级、处置状态。
批量操作：导出 CSV、标记已处理。
点击行展开详情或跳转地图定位。
高优先级（违规停放）红色标签置顶。
```

### 3.3 待人工审核 `/reviews`

```
待人工审核队列页。
顶部统计：待审核总数、今日新增、超时未处理。
列表卡片式，每条含：
  - 车牌（或「未识别」）
  - 原因（OCR失败/置信度低/车牌污损）
  - 拍摄时间、航点、设备
  - 证据截图（可放大）
  - AI 判定建议
  - 操作：确认车牌并补录 / 标记误报 / 加入白名单 / 标记外来车
支持键盘快捷：确认/跳过。
审核后从队列移除，写入 review_status。
```

### 3.4 白名单管理 `/whitelist`

```
白名单管理页。
支持 CSV 批量导入 + 单条新增。
表格：车牌、车主、楼栋、车位、车辆类型（私家车/访客/commercial）、有效期、操作。
导入预览：成功/失败条数、格式错误提示。
与立项一致：vehicle_type = private/visitor/commercial。
```

### 3.5 报告中心 `/reports`

```
报告中心页。
按任务列出已生成报告，卡片含：任务ID、日期、设备、违规数、外来车数。
支持 HTML 预览、CSV 下载、截图 ZIP 下载。
可按周/月汇总趋势小图（简单柱状图即可）。
```

### 3.6 系统设置 `/settings`

```
系统设置页，Tab 分组：
- 路线与航点：编辑 patrol_waypoints.yaml 可视化表单
- 禁停区管理：与地图页联动，列表+编辑
- 告警规则：置信度阈值、去重时间窗口
- 连接配置：默认 Bridge、超时参数
```

## 4. 前端 React 开发提示词

```
在现有 web-car-app（React18+Vite+TS）上扩展巡牌云管理端：
1. 新增 AppShell 布局（Sidebar+Topbar+Outlet）
2. 复用 styles.css 变量，不引入新 UI 库
3. 设备管理：localStorage + API 双写，列表持久化
4. 控制台：拆分 LiveMap + 复用 RemotePage 的 ButtonPad/Joystick/VideoPanel
5. 实时位姿：订阅 pose_update，50~100ms 节流渲染
6. 大地图：支持绘制多边形（禁停区），违规 Marker 点击弹 Drawer
7. 巡逻记录：表格 + 详情 Tab

9. 路由守卫：/console 需已选 device_id，否则跳转 /fleet
10. 所有文案中文，状态用 Tag 组件区分颜色
```

## 5. 后端 API 契约摘要

详见 `patrol-api/` 实现与 `schema.sql`。

| 模块 | 端点 |
|------|------|
| 设备 | `GET/POST /api/devices` · `PUT/DELETE /api/devices/:id` · `POST .../connect` · `GET .../status` · `GET .../pose` |
| 地图 | `GET /api/map` · `GET /api/map/waypoints` · `GET/POST/PUT/DELETE /api/map/zones` |
| 巡检 | `POST /api/patrol/start` · `POST /api/patrol/stop` · `GET /api/patrol/status` · `GET /api/patrol/tasks` · `GET .../:id` · `GET .../events` · `GET .../report` |
| 违规 | `GET /api/violations` · `GET /api/violations/:event_id` |
| 审核 | `GET /api/reviews/pending` · `POST /api/reviews/:event_id/resolve` |
| 白名单 | `GET /api/whitelist` · `POST /api/whitelist` · `POST /api/whitelist/import` |
| 工作台 | `GET /api/dashboard/summary` |
| WS | `ws://host/patrol/live` → `pose_update` / `patrol_status` / `patrol_event` / `device_status` / `violation_alert` |
